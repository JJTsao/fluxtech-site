# Fluxtech 品牌網站規格

狀態：首版草案（V1）

## 1. 專案目標

- 建立一頁式靜態品牌網站，向各產業的中小企業、組織與業主介紹 Fluxtech。
- 核心訴求：輕量、快速上線、合理預算、不強制綁定維護月費。
- 主要轉換目標：引導訪客加入 LINE 官方帳號諮詢。
- 品牌感受：科技、前瞻、敏捷；「流動」代表快速開發、部署與上線。

## 2. 首版範圍

### 包含

- 一頁式響應式網站。
- 淺色／深色模式：預設跟隨系統，可手動切換並記住選擇。
- 作品視覺縮圖、簡介與外部網站連結。
- App 品牌視覺與「即將推出」狀態。
- LINE 官方帳號 CTA。
- Dummy data，保留後續替換正式資料的結構。
- GA4 與重要點擊事件。
- 技術 SEO 與 Google Search Console 基礎設定。
- 預留未來加入公開方案價格的版面與資料結構。

### 不包含

- CMS、會員、電商或客製後台。
- 公開價格方案。
- 站內案例詳情頁、部落格與多語系。
- 長期內容 SEO、廣告投放或成效儀表板。

## 3. 頁面結構

1. 導覽列：區塊錨點、主題切換、LINE 諮詢。
2. Hero：品牌定位、核心價值、主要與次要 CTA。
3. 服務：一頁式形象／招生網站、多頁或產品型網站、後台需求另行報價。
4. 作品案例：縮圖、產業、摘要、服務標籤與「查看網站」。
5. App 預告：品牌視覺、簡介與「即將推出」。
6. 合作流程：需求訪談、規劃提案、開發、上線交付。
7. FAQ：報價方式、維護費、內容修改、後台與交付範圍。
8. LINE CTA：再次強調免費諮詢。
9. Footer：公司資訊、聯絡方式、社群與政策連結。

## 4. 內容與案例規則

- 招生網站視為一頁式形象網站的產業應用，不列為獨立服務。
- 「不收維護費」統一表述為「不強制綁定維護月費」；後續修改與支援依需求另議。
- 尚未簽約或未獲公開授權的提案，不得呈現為已完成客戶案例。
- 案例資料至少包含：`id`、名稱、產業、摘要、服務標籤、縮圖、網址、狀態。
- 案例外連以新分頁開啟，並加上安全屬性。

## 5. 視覺與素材

- 品牌主色：Flux Lime `#C8F500`；淺色模式文字用深綠，避免螢光綠直接作為小字。
- 深色模式使用帶綠調的深灰，不使用大面積純黑。
- Logo 應準備淺色與深色背景版本，優先使用透明背景檔案。
- Hero、光軌、科技紋理與抽象點綴優先評估生圖素材。
- 不使用 inline SVG 製作裝飾；簡單線條、漸層與狀態可使用 CSS。
- 生圖素材輸出 WebP／AVIF，提供響應式尺寸；首屏圖片優先載入，其餘延遲載入。
- 動效保持克制，並支援 `prefers-reduced-motion`。

## 6. GA4 量測

| 事件 | 觸發時機 | 主要參數 |
|---|---|---|
| `line_consult_click` | 點擊任一 LINE 諮詢 CTA | `placement` |
| `portfolio_click` | 點擊作品外部連結 | `case_id`、`category`、`destination` |

- `line_consult_click` 設為 GA4 重要事件。
- `placement` 至少區分 `nav`、`hero`、`footer`。
- 保留 GA4 自動收集的頁面瀏覽、捲動與外部連結事件。
- 使用正式 Measurement ID 後，在 GA4 DebugView 驗證事件與參數。

## 7. 技術 SEO 與 Search Console

- 語意化 HTML、單一 H1、合理標題層級與可爬取連結。
- 設定 title、meta description、canonical、favicon、Open Graph 與分享縮圖。
- 建立 `robots.txt`、`sitemap.xml` 與 `Organization` JSON-LD。
- 圖片提供描述性檔名、替代文字與正確尺寸。
- 上線後驗證 Search Console 網域、提交 sitemap，並檢查首頁索引狀態。
- 首版 SEO 以可被理解、索引及正確分享為目標，不承諾特定關鍵字排名。

## 8. 驗收條件

- 手機、平板與桌面版內容完整，無水平溢出。
- 淺／深色模式載入時不閃爍，重新整理後保留使用者選擇。
- 鍵盤操作、焦點樣式與文字對比符合 WCAG AA 基本要求。
- LINE、案例與導覽連結正確，無阻斷性 Console 錯誤。
- 生圖與案例圖片經壓縮，不明顯拖慢首屏載入。
- GA4 重要事件可在 DebugView 驗證。
- SEO 檔案、metadata、結構化資料與分享縮圖通過基本檢查。

## 9. 上線前待提供

- LINE 官方帳號連結。
- GA4 Measurement ID。
- 正式網域、部署位置與 DNS 配合方式。
- 正式 Logo、公司資料、聯絡方式及案例素材。
- Search Console 使用的 Google 帳號或網域驗證配合。
